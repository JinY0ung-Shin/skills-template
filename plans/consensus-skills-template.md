# Skills Template - Consensus Plan

> Generated by /cplan (Claude + Codex consensus)
> Consensus reached: Round 3

## Requirement

누구든 이 레포만 보면 Claude Code skill을 올바르게 만들 수 있는 공용 템플릿 레포

## Agreed Plan

### 목표
> Claude Code skills는 [Agent Skills](https://agentskills.io) 오픈 표준을 따르며, Claude Code 전용 확장 필드를 추가로 지원한다.
> Skills는 기존 `.claude/commands/` 시스템을 통합·대체한 상위 시스템이다. SKILL.md 하나에 frontmatter로 모든 제어(인자, 모델, 도구 권한, 서브에이전트, hooks 등)를 선언할 수 있다.

## Implementation Steps

1. **예제 스킬 6개 작성** {parallel: true}
   - Actions: review, gen-test, pr-summary, deploy, legacy-context, secure-ops 각각의 SKILL.md + 보조 파일 작성
   - Files: `.claude/skills/review/SKILL.md`, `.claude/skills/gen-test/SKILL.md`, `.claude/skills/pr-summary/SKILL.md`, `.claude/skills/pr-summary/review-checklist.md`, `.claude/skills/pr-summary/scripts/collect-metrics.sh`, `.claude/skills/deploy/SKILL.md`, `.claude/skills/legacy-context/SKILL.md`, `.claude/skills/secure-ops/SKILL.md`, `.claude/skills/secure-ops/scripts/security-check.sh`
   - Dependencies: none

2. **템플릿 4개 작성** {parallel: true}
   - Actions: basic.md, with-tools.md, with-hooks.md, advanced.md 빈 템플릿 작성
   - Files: `templates/basic.md`, `templates/with-tools.md`, `templates/with-hooks.md`, `templates/advanced.md`
   - Dependencies: none

3. **REFERENCE.md 작성** {parallel: false}
   - Actions: 필드 레퍼런스([core]/[ext]/[CLI-only]/[legacy] 라벨), Invocation Control Matrix, Progressive Disclosure, 이름 규칙, 저장 위치/우선순위(Enterprise > Personal > Project, Plugin 별도), Skills vs Commands vs Agents, Description 가이드, 검증/트러블슈팅, 보안, 크로스 툴 호환성, FAQ
   - Files: `REFERENCE.md`
   - Dependencies: Step 1 (예제 참조)

4. **README.md 작성** {parallel: false}
   - Actions: Quick Start, 구조 설명, 예제 안내, frontmatter 요약 테이블, 변수/동적 컨텍스트, 설치 방법, 명명 규칙, 보안 경고, 크로스 툴 호환성 노트
   - Files: `README.md`
   - Dependencies: Step 1, 2, 3

5. **커밋 & 푸시** {parallel: false}
   - Actions: git add, commit, push
   - Dependencies: Step 1, 2, 3, 4

## Acceptance Criteria

- [ ] 템플릿 4개가 `templates/` 하위에 주석 포함 빈 틀로 존재

- [ ] 모든 예제/템플릿에 name, description 명시 (크로스 툴 호환성)

- Storage rules correctly described (Enterprise > Personal > Project, Plugin namespace separate)
- Well-balanced documentation scope (README + REFERENCE + templates + examples)
- Cross-tool compatibility note is a strong maintainability choice

**Overall:** Plan is solid enough to approve. Recommend using official Claude Code and Agent Skills docs as normative citations, treating blog posts as supplemental only.
